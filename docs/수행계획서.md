# LMS 수강신청/마이페이지 기능 개선 및 클라우드 서비스 구축 수행계획서

## 1. 과업 개요
본 과업은 LMS 웹서비스의 수강신청 흐름과 마이페이지/시간표 조회 UX를 개선하여 사용자가 신청 → 내역 확인 → 취소 → 반영 확인까지 한 화면 내에서 직관적으로 처리할 수 있도록 하는 것을 목표로 한다.
또한 클라우드 환경에서 운영 가능한 LMS 구조를 고려해 서버/DB/API 중심의 확장 가능한 아키텍처를 지향한다.

## 2. 수행 목표
### 2.1 기능 목표
- 강의 데이터 재구성(직군/직급/주야)
- 정원 마감 시 신청불가 + 인원표기(현재/최대)
- 수강신청 내역 동기화(신청 시 추가, 취소 시 제거)
- 시간표 조회: 신청 강의의 강의명/교수/강의시간 반영
- 상태 저장: 서버 기반 신청정보 저장 및 조회
- 시간표 중복 검증: 동일 시간대 강의 신청 차단

### 2.2 UI/UX 목표
- 화이트/블루 계열 UI
- 탭 분리(개설과목 조회/신청, 수강신청 내역, 시간표 조회)
- 과목코드/분반 빠른 신청
- 피드백 메시지(성공/취소/정원초과/시간중복)

## 3. 시스템 아키텍처 구성
### 3.1 API 서버
- Spring Boot 기반 API 서버 (AWS EC2/Beanstalk 배포 가능)

### 3.2 데이터베이스
- 1안: AWS RDS(MySQL/PostgreSQL)
- 2안: Supabase(PostgreSQL + Auth)

### 3.3 프론트엔드
- Thymeleaf SSR + JavaScript 동기화
- 시간표는 신청 내역 기반 동적 렌더링

## 4. 구현 범위
### 4.1 백엔드
- HomeController: 페이지 데이터 전달
- EnrollmentController: 신청/취소/목록 REST API
- 시간중복 및 정원 검증 로직

### 4.2 프론트엔드
- apply.html 내 신청/취소 토글
- 신청내역/시간표 동기화
- 서버 API(`/enroll/apply`, `/enroll/cancel`, `/enroll/list`) 연동

### 4.3 보안/운영
- 로그인 사용자 기반 신청 제한(차기)
- CSRF/보안그룹 설정(차기)

## 5. 상세 수행 절차
1) 강의 데이터 모델 정비
2) 신청/취소 API 구현
3) 시간중복 검증 추가
4) 신청내역/시간표 UI 연동
5) 서버 실행 및 통합 테스트
6) 클라우드 배포 준비

## 6. 테스트 계획
### 6.1 기능 테스트
- 신청 가능 강의 신청
- 신청내역 반영
- 시간표 반영
- 취소 후 내역/시간표 제거
- 동일 시간 강의 신청 차단

### 6.2 예외 테스트
- 정원 초과 신청
- 없는 과목코드/분반 신청
- 서버 재시작 후 사용자 내역 조회

## 7. 산출물
- 소스코드

- 신청부터 확인/취소까지 일관된 사용자 경험 제공
- 시간 중복 방지로 학습 일정 관리 효율 향상
