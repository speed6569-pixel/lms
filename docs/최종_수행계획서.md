# 기업·사업장 대응 역량 강화를 위한 LMS 구축 수행계획서 (최종 제출용)

## 1. 과업 개요
본 과업은 기업, 마트 및 각종 사업장에서 근무하는 신입/재직자를 대상으로,
업무 수행에 필요한 매뉴얼·프로그램 사용법·장비 운용 지식과 더불어
재난/특수상황 대응 역량을 체계적으로 학습할 수 있는 통합 LMS를 구축하는 것을 목표로 한다.

특히 화재, 정전, 안전사고, 고객 응대 이슈, 시스템 장애 등 현장 리스크를 고려하여
직군·직급별 필수 교육과 대응 매뉴얼을 결합한 실무형 학습 체계를 제공한다.

---

## 2. 추진 목표

### 2.1 기능 목표
1. **직군·직급 기반 교육 체계**
   - 직군: 개발 / 기획·마케팅 / 경영 / 영업
   - 직급: 사원 / 주임 / 대리 / 과장 / 차장 / 부장
   - 신입 공통 과정 + 직무 심화 + 관리자 대응 과정 분리

2. **수강신청·취소·이수 관리**
   - 과정 조회, 신청/취소, 상태 반영
   - 정원 초과 시 신청 차단
   - 동일 시간대 중복 신청 차단

3. **시간표/일정 관리**
   - 신청된 강의 기준 시간표/내역 확인
   - 신청/취소 즉시 반영

4. **학습자료 제공**
   -  영상자료

5. **사용자별 학습 이력 관리**
   - 사용자 단위 신청 이력/이수 상태 조회

### 2.2 시스템 목표
- 웹 기반 통합 학습 시스템
- 서버 API 중심 구조 확보 (확장 가능한 아키텍처)
- 클라우드 운영 가능한 배포 구조 설계

---

## 3. 범위 정의 (MVP / 확장)

### 3.1 1차 구축(MVP)
- 수강신청/취소 API
- 정원 검증, 시간 중복 검증
- 수강신청 내역 조회
- 시간표 조회(신청 강의 기준)
- 직군/직급/주야 필터 기반 과정 조회

### 3.2 2차 확장
- 관리자 과정 등록/수정/삭제
- 필수과정 자동 지정 엔진
- 교육 통계 대시보드
- 재난대응 이수율 리포트

---

## 4. 시스템 아키텍처

### 4.1 백엔드
- Spring Boot (MVC + REST API)
- 주요 API
  - `POST /enroll/apply`
  - `POST /enroll/cancel`
  - `GET /enroll/list`

### 4.2 프론트엔드
- Thymeleaf 기반 화면 렌더링
- JavaScript 기반 실시간 필터(새로고침 없는 AND 필터)
- 탭 UI: 개설과목 조회/신청, 수강신청 내역, 시간표 조회

### 4.3 데이터 저장
- 현재: 서버 메모리 기반(세션 단위) MVP
- 확장: AWS RDS / Supabase(PostgreSQL) 연동

### 4.4 클라우드(운영안)
- EC2(or Elastic Beanstalk) + RDS + S3 + CloudFront
- Route53 + ACM(SSL)
- CloudWatch 로그/알람

---

## 5. 도메인/데이터 모델 (권장)
- `users` (사용자)
- `organizations` (기업/사업장)
- `courses` (과정)
- `course_sessions` (일정/정원/강의실)
- `enrollments` (신청 이력)
- `completions` (이수 상태)
- `learning_materials` (PDF/영상/체크리스트)
- `incident_playbooks` (재난 대응 매뉴얼)

---

## 6. 권한 체계 (RBAC)
- USER: 과정 조회/신청/취소/이수확인
- MANAGER: 팀 단위 이수현황 조회
- ADMIN: 과정/정원/일정 관리, 필수과정 지정
- SUPER_ADMIN: 시스템 설정/권한 관리

## 8. 테스트 계획

### 8.1 기능 테스트

- 신청/취소 반영 정확성

## 9. 비기능 요구사항(NFR)
- 응답속도: 핵심 API 95% 요청 500ms 이내
- 가용성: 월 99.5% 이상
- 감사로그: 신청/취소/이수변경 로그 저장
- 백업: 일일 백업 + 보관 정책 운영
